## 板块一：竞品深度拆解

### 🃏 1.1 Balatro — 为什么让人上瘾？

**核心 Loop 拆解（一局完整流程）：**

```
选牌组(Deck) → [Ante 1] → 小盲注 → 大盲注 → Boss盲注 → 商店 → 
                [Ante 2] → 小盲注 → 大盲注 → Boss盲注 → 商店 →
                ...重复至 Ante 8（共 8 轮×3 关 = 24 关战斗）
```

**每个战斗关的微循环（30秒级）：**
1. 拿到 8 张手牌
2. 可弃牌重抽（最多 3–5 次换牌）
3. 选牌打出扑克牌型（对子/顺子/同花等）
4. 系统计算：`基础筹码 × 倍率 = 得分`
5. 达到目标分数过关，否则失败

**"再来一局"的魔力来源（具体机制层面）：**

| 魔力机制 | 具体实现 | 心理学原理 |
|---------|---------|-----------|
| **Joker 组合爆发** | 150+ 张 Joker，每张改变一条规则。如 "所有红色牌 +3 倍率" + "每回合手牌数 +2" 叠加出天文数字得分 | **涌现式满足**：玩家发现设计师没预料到的组合，产生强烈的"我真聪明"感 |
| **Boss 盲注变异** | 每个 Boss 有独特规则修改器（如"所有方块牌翻面"、"每手牌只能出 4 张"），迫使玩家改变策略 | **适应性挑战**：打破舒适区，制造"这次不同"的新鲜感 |
| **商店随机性** | 每次商店只出现 2 张随机 Joker + 1 个 Booster Pack + 1 张 Voucher，你永远不知道会遇到什么 | **可变奖励时间表**（Variable Ratio Reinforcement）：赌博心理的核心机制 |
| **分数指数爆炸** | 倍率系统是乘法而非加法。2×3×5 = 30 倍。一手牌可以从 100 分暴涨到 100,000 分 | **权力幻想**：玩家感受到"我在打破游戏"的兴奋 |

**内容深度保证重玩性：**

| 内容维度 | Balatro 数量 | 斗破乾坤当前 | 差距倍数 |
|---------|-------------|------------|---------|
| Joker | **150+** | 20 | **7.5×** |
| 可用牌组(Deck) | **15** | 0（无概念） | **∞** |
| Boss 盲注变异 | **~25种** | 0 | **∞** |
| 商店道具(Voucher) | **32** | 0 | **∞** |
| Booster Pack 类型 | **6大类** | 0 | **∞** |
| 行星牌(Planet) | **12** | 0 | **∞** |
| 塔罗牌(Tarot) | **22** | 0 | **∞** |
| 幽灵牌(Spectral) | **18** | 0 | **∞** |
| 总关卡数 | **24战斗+8商店** | 6战斗+2商店 | **4×** |

**Balatro 的解锁进度系统：**
1. **牌组解锁**：通关解锁新牌组，每个牌组有独特规则（如"所有牌都是方块"、"起始只有 26 张牌"）
2. **Joker 发现**：在游戏中遇到新 Joker 自动记录到图鉴，激励探索
3. **赌注等级**：每通关一次某牌组，解锁该牌组的更高赌注（类似 Ascension），共 8 级
4. **挑战模式**：通关后解锁特殊挑战（固定条件，如"只能用一种 Joker"）

> **🔑 核心洞察**：Balatro 的上瘾不来自"更多关卡"，而来自 **Joker 之间的组合涌现**。150 张 Joker 的组合空间是天文数字——每局都是独特的 Build，这才是重玩性的根基。

### 🌙 1.2 月圆之夜 — 中国市场成功密码

**核心玩法 Loop：**
```
选角色 → 选卡组 → 进入章节地图（分支路线选择）→ 
战斗（回合制卡牌战斗，消耗行动力打牌）→ 奖励（选新卡/升级卡/金币）→ 
事件节点（随机事件/商店/精英怪）→ Boss → 下一章节
```

**IAP 扩展包模式详解：**

| 扩展包 | 价格 | 内容 |
|--------|------|------|
| 基础版 | ¥1（引流价） | 1 个角色（小红帽）+ 第一章 |
| 角色包：女巫 | ¥12 | 全新角色 + 专属卡组 + 新机制（药剂系统） |
| 角色包：魔镜 | ¥12 | 全新角色 + 反射/镜像机制 |
| 章节包：狼人杀 | ¥18 | 新章节 + 新 Boss + 新卡池 |
| 章节包：小美人鱼 | ¥18 | 海底主题新章节 |
| 大型DLC：药剂师 | ¥25 | 全新故事线 + 新职业 + 新机制 |
| **总花费** | **~¥100+** | 远超单次买断 |

**关键商业智慧**：
- ¥1 入门价消除付费心理门槛（"只要 1 块钱试试"）
- 每个扩展包是**独立的内容增量**，不是 P2W
- 角色包 = 全新玩法体验 ≠ 数值变强。玩家为**"新鲜感"**付费，而非为**"变强"**付费
- 这正是斗破乾坤应该学的模式

**留存机制：**
- **每日挑战**：固定种子的每日一局，有排行榜
- **角色差异化**：每个角色玩法完全不同，等于 N 个游戏
- **噩梦难度**：通关后解锁更高难度，等于翻倍内容
- **成就系统**：大量成就引导探索

### ⚔️ 1.3 Slay the Spire — 长期留存密码

**每局不同的关键变量数量：**

| 随机变量 | 变化空间 | 说明 |
|---------|---------|------|
| 卡牌池抽取 | ~75 张职业卡中随机 | 每次奖励 3 选 1 |
| 遗物获取 | ~170 个遗物随机 | 每次 Boss 奖励 3 选 1 |
| 地图路线 | 3-4 条分支路线 | 玩家选择战斗/事件/精英/商店的顺序 |
| 随机事件 | ~70 个事件 | 各有独特选择和后果 |
| 药水掉落 | ~30 种药水 | 战斗中的一次性资源 |
| 敌人出现顺序 | ~50 种敌人组合 | 每层地图随机分配 |
| Boss 类型 | 每层 3 个中抽 1 个 | 需要不同应对策略 |
| **组合空间** | **理论上无穷** | 每局的 Build 路径独一无二 |

**Ascension（挑战等级）系统工作原理：**

```
Ascension 0（基础）→ 通关解锁 A1
A1: 精英怪 HP +? → 通关解锁 A2  
A2: 普通怪增加 1 个 buff → 通关解锁 A3
...
A15: 出第一张牌前，Boss 获得额外能力
...
A20: 所有 debuff 叠满，终极挑战
```

**每一级只增加一条小规则**，但叠加起来天差地别。这个系统的精妙之处：
- **零内容开发成本**：不需要新卡牌/新敌人/新地图，只需修改数值和规则
- **极长尾巴**：21 个等级 × 4 个角色 = 84 个需要通关的挑战
- **社区荣誉**：A20 通关是硬核玩家的勋章
- **适用于 Solo 开发者**：实现极简，效果极强

> **🔑 核心洞察**：Slay the Spire 证明了 **Ascension 系统是零成本最大化留存的终极武器**。斗破乾坤必须有这个。

### 🪝 钩子 1：「沉没成本 + 内容断崖」— 试玩到成瘾再收费

| 维度 | 设计 |
|------|------|
| **心理学原理** | 禀赋效应 + 损失厌恶。玩家在 3 关中已投入时间构筑了 Build（选了 Joker、买了 Buff），"放弃 = 亏了"。IKEA 效应：自己组装的东西更有价值。 |
| **当前问题** | DemoGate 出现在第 3 关结束后，此时玩家可能还没体验到 Joker 组合的"爽点"（商店在第 4 关才出现！） |
| **优化方案** | ① 将第一个商店从第 4 关移到第 3 关（让试玩用户体验到购买 Joker 的快感）<br>② 第 3 关结束时展示"你还差 X 步就能解锁 Boss 关"而不是直接弹付费墙<br>③ DemoGate 展示玩家当前 Build 的潜力："你的组合可以打到 50,000 分！" |
| **代码修改** | `FloorConfig.allFloors` 重排顺序：关1→关2→商店→关3→DemoGate |
| **预期影响** | 试玩→付费转化率 +30~50%（月圆之夜用 ¥1 实现了超高转化率，同理） |
| **实现难度** | ⭐ 极低（改 FloorConfig 数组顺序即可） |
| **优先级** | 🔴 P0 — 直接影响收入 |

### 🪝 钩子 2：「完成主义驱动」— 图鉴收集系统

| 维度 | 设计 |
|------|------|

| **实现难度** | ⭐⭐ 低（1-2 天） |
| **优先级** | 🟠 P1 |

### 📊 当前流程诊断

```
[HomeView] → 点"开始冒险" → [MapView] → 点"出发" → [BuildSelectView] → 选流派 → [BattleView]
   ↑                           ↑                        ↑
   问题1: 只有"开始冒险"       问题2: 地图纯展示         问题3: 6个流派
   一个按钮，没有上下文         没有交互意义              新手不知道选哪个
   "我该干什么？"              "看完了，然后呢？"         "选错了怎么办？"
```

**点击次数到开始游戏：4 次点击 + 1 次阅读选择**（太多了！）

### 竞品首屏分析

| 游戏 | 打开后第一个画面 | 到开始游戏的点击数 |
|------|----------------|------------------|
| **Balatro** | 主菜单 → "Play" → 选牌组 → 开始。**但是**：选牌组界面同时展示了解锁进度和牌组特殊规则，本身就是内容 | **2 次** |

| **Slay the Spire** | 主菜单 → 选角色（同时看到 Ascension 等级）→ 开始 | **2 次** |

### 🎯 重设计方案：「1 秒知道干什么，3 秒进入游戏」

**新首屏架构：**

```

│          🎴 斗破乾坤                   │
│                                      │
│  ┌──────────────────────────────┐    │
│  │  🔥 继续冒险（第5关/8关）      │ ← 有存档时显示，最大最醒目
│  │     当前Build: 炸弹狂人        │    1 秒直接回到战斗
│  │     Score: 1,350              │

│                                      │
│  ┌──────────────────────────────┐    │
│  │  ⚡ 快速开始                    │ ← 无存档时主按钮
│  │     随机流派，直接进入第1关      │    跳过所有选择步骤
│  └──────────────────────────────┘    │
│                                      │
│  ┌────────┐ ┌────────┐ ┌────────┐   │
│  │📋 选流派│ │📅 每日  │ │🏆 图鉴 │ ← 次要入口
│  │  挑战   │ │  挑战   │ │  成就  │
│  └────────┘ └────────┘ └────────┘   │
│                                      │

└──────────────────────────────────────┘
```

**核心变化：**

| 变化 | 原来 | 现在 | 理由 |
|------|------|------|------|
| 删除 MapView | Home → Map → Build → Battle | Home → Battle | 地图纯展示，无交互价值，白白增加一步 |

**代码层面修改：**
- `ContentView.swift`：移除 `.map` 状态，Home 直接到 `.buildSelect` 或 `.battle`
- `HomeView.swift`：重构为新首屏布局，增加存档检测和"快速开始"逻辑
- `RogueRun.swift`：增加 `startQuickRun()` 方法（随机选择流派）
- `GameSave.swift`：增加存档恢复到 `RogueRun` 的方法

**优先级**：🔴 P0 — 这是用户反馈的核心痛点
**实现难度**：⭐⭐ 低（2-3 天）

## 板块四：长期留存系统设计

---

### 🏔️ 系统 1：Ascension 挑战等级（零内容成本的留存倍增器）

**机制设计：**

```
挑战等级 0（基础）— 当前难度
挑战等级 1：每关目标分数 +15%
挑战等级 2：每关出牌次数 -1
挑战等级 3：商店物品价格 +20%
挑战等级 4：起始金币 -30
挑战等级 5：每关换牌次数 -1
挑战等级 6：敌人目标分数 +30%（叠加等级1）
挑战等级 7：Joker 最大携带数 -1（从 5 降到 4）
挑战等级 8：商店只出现 1 张 Joker（而非 2 张）
挑战等级 9：起始金币 -50（叠加等级4）
挑战等级 10：所有修改器叠加 — 终极挑战 🏆
```

**每个等级通关解锁下一级 + 独特成就徽章。**

**与现有代码集成：**

```swift
// RogueRun.swift 新增属性
@Published var ascensionLevel: Int = 0

// FloorConfig 新增方法
func adjustedForAscension(_ level: Int) -> FloorConfig {
    var target = targetScore
    var plays = maxPlays
    var discards = maxDiscards
    
    if level >= 1 { target = Int(Double(target) * 1.15) }
    if level >= 2 { plays -= 1 }
    if level >= 5 { discards -= 1 }
    if level >= 6 { target = Int(Double(target) * 1.30) }
    // ...
    return FloorConfig(floor: floor, name: name, 
                       targetScore: target, maxPlays: max(plays, 2),
                       maxDiscards: max(discards, 0), ...)
}
```

**需要修改的文件：**
- `RogueRun.swift`：所有 `currentFloor` 引用改为 `currentFloor.adjustedForAscension(ascensionLevel)`
- `FloorConfig`：新增 `adjustedForAscension()` 方法
- `HomeView.swift`：展示当前最高 Ascension 等级
- `Achievement.swift`：新增 10 个 Ascension 相关成就
- `PlayerStats`：新增 `highestAscension` 字段

**预期影响：**
- 内容寿命从 8 关 × 6 流派 = ~48 次变成 **8 关 × 6 流派 × 11 等级 = ~528 次**

### 🏴 Boss 关卡机制设计（核心差异化）

Boss 不只是"分数更高"——每个 Boss 有独特的**规则修改器**，打破玩家的固有策略：

| Boss | 关卡 | 规则修改器名称 | 效果 | 策略变化 |
|------|------|--------------|------|---------|
| **县令** | 8 | 📜 官府禁令 | **禁用一种随机牌型**（如本局不能出顺子） | 迫使玩家适应，不能只依赖一种流派 |
| **太子** | 13 | 👑 皇家特权 | **目标分数每次出牌后 +5%**（拖得越久越难） | 鼓励速战速决，惩罚慢热型 Build |
| **地主** | 15 | 🔥 双重压制 | **「官府禁令」+「分数衰减」**（每次出牌后，下一次出牌得分 -10%） | 终极考验：需要精确计算每一手牌 |

**额外 Boss 修改器池（随机从中抽取，每局不同）：**

| 修改器 | 效果 | 设计意图 |
|--------|------|---------|
| 🪞 镜像 | 所有牌型得分颠倒（单张最高，炸弹最低） | 完全颠覆策略 |
| 💀 死亡计时器 | 限时 60 秒完成本关 | 压力测试 + 肾上腺素 |
| 🎭 易容 | 每次出牌后，手牌中随机 2 张花色互换 | 打乱顺子/同花计划 |
| 🔒 封印 | 本关前 2 次出牌不计分 | 必须利用连击 Buff 在后期爆发 |
| 💰 贪婪税 | 每次出牌扣 10 金币 | 经济压力测试 |
| 🎯 精确打击 | 只有得分恰好 = 目标分数 ±10% 才算过关（超出太多也失败） | 精确控分挑战 |

**代码实现：**

```swift
// 新增 Engine/BossModifier.swift

    case bannedPattern    // 禁用一种牌型
    case escalating       // 目标递增

    case timeLimit        // 限时
    case sealed           // 前N次不计分

    case precisionOnly    // 精确打击
}

// FloorConfig 扩展
struct FloorConfig {
    // ... existing fields ...
    let bossModifiers: [BossModifier]  // 空 = 非Boss关
}
```

### 🎲 变量随机化设计（同层级不同体验）

**当前问题**：每次第 1 关都是"乡野牌局，目标 200 分"——零变化。

**随机化层级：**

| 随机层 | 具体实现 | 效果 |
|--------|---------|------|
| **关卡修改器** | 每关随机附加 0-1 个修改器（如"本关顺子 +50%"、"本关换牌次数 +1"） | 同一关每次体验不同 |
| **商店内容** | ✅ 已实现（随机 2 Joker + 2 Buff） | 保持 |
| **目标分数波动** | 目标分数 ±15% 随机浮动 | 微观不确定性 |
| **手牌质量** | 已实现（随机发牌） | 保持 |
| **Boss 修改器** | Boss 关从池中随机抽 1 个修改器 | 每次 Boss 战不同 |
| **关卡事件** | 非战斗非商店的新节点类型：随机事件（3选1选择题，影响游戏状态） | 增加路线多样性 |

**随机事件示例（新增节点类型）：**

```markdown

选项 A: 买入 — 花 80 金币，获得"下一关所有得分 ×2"
选项 B: 卖出 — 卖掉 1 个已装备的 Joker，获得其价格的 150%
选项 C: 闲逛 — 发现 1 个未见过的 Joker 加入图鉴（但不获得）

```
