### 1.2 主流商业模式（均为 F2P 免费+内购）

- **虚拟货币售卖**（核心收入）：钻石/元宝，¥1~¥328 不等
- **订阅会员**（持续收入）：JJ ¥30/月、途游 ¥18/月
- **赛事体系**：排位赛/锦标赛/竞技场，促活+刺激消费
- **装扮/皮肤系统**：英雄卡、时装、牌桌特效、牌面框
- **抽奖箱/概率道具**：gacha 机制
- **副玩法生态**：农场家园、休闲小游戏，提升留存

### 1.3 市场特点

- **极度红海**：腾讯占据绝对头部（282万评分），头部3家瓜分大部分市场
- **产品趋势**：单一斗地主 → 棋牌合集平台（掼蛋、麻将等多玩法聚合）
- **年龄分级**：均为 18+（模拟赌博标签）
- **海外市场空白**：美区搜 "doudizhu" 仅 9 个结果，头部产品仅 5 个评分

### 1.4 结论

正面挑战头部不可行。个人开发者需要找差异化切入点。

## 三、选定方向：Roguelike 斗地主（类 Balatro）

### 3.1 灵感来源

Balatro（单人扑克 Roguelike）2024年由一个人开发，收入超5000万美元。
它用的是西方扑克规则，而 **中国牌型体系（炸弹/飞机/火箭）完全没人做过**。

### 3.2 核心玩法构想

- **单人闯关 + 构筑卡组**的 Roguelike
- 每局随机发牌，通过打出牌型（顺子/炸弹/飞机）得分
- 收集"小丑牌"/"特殊牌"改变规则（如：所有3变成炸弹、顺子得分×3）
- Roguelike 层层递进，Boss 关是超强 AI 地主
- 可融入中国文化主题（地主/农民/土地革命叙事包装）

### 3.3 商业模式

- **买断制 ¥25~45**（或 $4.99~$6.99）
- 不依赖内购和持续运营
- 可同时上 iOS + Steam
- 后续可出 DLC 扩展包（新角色/新规则/新关卡）

### 3.4 核心优势

1. Balatro 已验证品类天花板极高
2. 中国牌型体系是全新设计空间，无竞品
3. 单人游戏 = 不需要服务器/匹配/运营
4. 美术可以像素风/极简风，一人可搞定
5. 全球发行（中国牌型对海外玩家也有新鲜感）
6. 开源斗地主 AI（DouZero/RLCard）可作为 Boss AI 基础

### 3.5 风险与挑战

- 斗地主牌型系统比扑克复杂，设计 Roguelike 构筑的平衡难度更高
- 需要让不懂斗地主的海外玩家也能快速上手（教学设计很重要）
- 买断制在国内移动端接受度低，主战场可能是 Steam + 海外 iOS

## 四、备选创意（存档）

- **叙事斗地主「牌局人生」**：打牌推动剧情，输赢改变故事走向
- **AI 复盘教练**：类 Chess.com，AI 分析每手牌最优解，ELO 评分系统
- **角色技能斗地主**：每角色有独特技能改变规则（类杀戮尖塔）
- **海外精品斗地主**：面向北美/东南亚华人，中英双语，无氪金
- **斗地主 × 大富翁**：赢牌赚钱买地，输牌破产
- **AR 斗地主**：扫描实体扑克牌，AI 提示最优出牌
- **反向斗地主**：目标是最后出完牌的人赢（策略反转）

## 五、技术选型

> 开发者背景：Swift/SwiftUI 熟练，无游戏开发经验，先做 iOS，国潮水墨风

### 5.1 方案对比

| 方案 | 框架 | 语言 | 优势 | 劣势 |
|------|------|------|------|------|
| **A. SpriteKit + SwiftUI ✅** | Apple 原生 | Swift | 零学习成本、与 SwiftUI 无缝混合 | 仅限 Apple 平台 |
| B. Unity | Unity 引擎 | C# | 跨平台一步到位 | 需学 C#、包体大 |
| C. Godot | Godot 引擎 | GDScript | 免费开源、跨平台 | 需学新引擎 |

### 5.2 选定方案：SpriteKit + SwiftUI

- SwiftUI 做菜单/UI（商店、地图、背包），SpriteKit 做牌桌场景
- 两者通过 `SpriteView` 在同一项目中混合使用
- 如果验证成功，后续用 Godot/Unity 重写 Steam 版

### 5.3 架构设计

```
┌─────────────────────────────────────────┐
│              App Layer                  │
│        SwiftUI (菜单/地图/商店)          │
│            ┌──────────┐                 │
│            │SpriteView│                 │
│            └────┬─────┘                 │
│        SpriteKit (牌桌场景)              │
├─────────────────────────────────────────┤
│            Game Logic                   │
│  ┌──────────┐ ┌──────────┐ ┌────────┐  │
│  │CardEngine│ │RogueCore │ │ AI Bot │  │
│  │ 牌型判定 │ │回合/关卡  │ │对手AI  │  │
│  │ 出牌规则 │ │奖励/构筑  │ │难度分级│  │
│  └──────────┘ └──────────┘ └────────┘  │
├─────────────────────────────────────────┤
│            Data Layer                   │
│  SwiftData (存档/进度) + JSON (配置表)   │
└─────────────────────────────────────────┘
```

### 5.4 关键技术栈

| 模块 | 技术 | 说明 |
|------|------|------|
| UI框架 | SwiftUI | 菜单、地图、商店、设置 |
| 游戏渲染 | SpriteKit | 牌桌场景、卡牌动画、特效 |
| 数据持久化 | SwiftData | 存档、解锁进度、统计 |
| 配置数据 | JSON | 关卡配置、牌型得分表、Buff 配置 |
| 音频 | AVFoundation | BGM + 音效 |
| 水墨特效 | SKShader + Metal | 自定义 shader 做墨水晕染 |
| AI 对手 | 纯 Swift | 规则AI → MCTS → CoreML(DouZero) |
| 内购 | StoreKit 2 | 买断制付费 |

### 5.5 AI 对手方案

| 难度 | 算法 | 说明 |
|------|------|------|
| 简单 | 规则引擎 | 最大牌优先出，基础策略 |
| 中等 | 启发式搜索 | 考虑剩余手牌结构，拆牌决策 |
| 困难(Boss) | MCTS | 蒙特卡洛树搜索 |
| 地狱(隐藏Boss) | CoreML | DouZero 模型转 CoreML 导入 |
